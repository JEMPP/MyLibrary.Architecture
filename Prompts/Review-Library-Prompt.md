# Review Library Prompt

## Architekturreferenz

Verwende die Architekturstandards aus diesem Repository.

Lies zunächst:

```text
Architecture-AI-Prompt.md
```

Diese Datei stellt die primäre Zusammenfassung der Architektur dar.

Verwende anschließend bei Bedarf:

```text
Standards/*
Templates/*
```

Falls Widersprüche auftreten, gilt folgende Priorität:

1. Architecture-AI-Prompt.md
2. Standards/*
3. Templates/*

---

## Aufgabe

Analysiere die hochgeladene Bibliothek und bewerte sie gegen die Standards aus MyLibrary.Architecture.

---

## Ziel

Ziel ist eine Architektur-, Qualitäts- und Wartbarkeitsbewertung.

Es sollen keine Refactorings durchgeführt werden.

Es sollen keine neuen Klassen erzeugt werden.

Es sollen keine umfangreichen Codeänderungen vorgeschlagen werden.

---

## Analyse

Erstelle zunächst eine Übersicht:

* Solution-Struktur
* Projektstruktur
* Namespaces
* Models
* Services
* Repositories
* Konfiguration
* Tests
* Logging
* Datenbankzugriffe
* externe Abhängigkeiten

---

## Bewertung

Bewerte die Bibliothek hinsichtlich:

* Architektur
* Wartbarkeit
* Testbarkeit
* Sicherheit
* Wiederverwendbarkeit
* Erweiterbarkeit
* Dokumentation

---

## Standards-Check

Prüfe insbesondere:

* Development Standards
* Database Standards
* Security Standards
* Testing Standards
* Naming Conventions
* GitHub Standards

---

## Abweichungen

Identifiziere:

* Verstöße gegen Standards
* technische Schulden
* Sicherheitsrisiken
* fehlende Tests
* Architekturprobleme
* Naming-Probleme

---

## Priorisierung

Ordne die Ergebnisse ein:

### Priorität 1

Kritische Probleme

### Priorität 2

Wichtige Verbesserungen

### Priorität 3

Optionale Verbesserungen

---

## Erwartete Ausgabe

Liefere:

1. Kurzbewertung
2. Stärken
3. Schwächen
4. Abweichungen vom Standard
5. Risiken
6. Priorisierte Maßnahmenliste

---

## Wichtige Regel

Keine Refactorings durchführen.

Keine umfangreichen Codeänderungen erzeugen.

Ziel ist ausschließlich die Bewertung und Vorbereitung eines späteren Refactorings.
